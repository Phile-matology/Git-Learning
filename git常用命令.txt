git status 查看仓库状态
返回是否为Git仓库，当前分支，是否有未追踪文件，是否有新追踪文件需提交

git init
初始化或重新初始化为Git仓库

git add 文件名
将未追踪文件添加至git仓库

git commit -m "提交备注"
将已添加文件提交至仓库并注释提交备注

git log
打印Git仓库提交日志
提供信息：commit编号，作者，时间，备注
输入字母q退出

git branch（+ 分支名称）
查看Git仓库分支情况
返回所有分支，当前分支前有*
（新建分支）

git checkout （-b）分支名称
切换到新分支
（在当前分支下创建新的分支，并切换至新分支）

git merge （被合并分支名）
合并分支至当前分支
会返回合并了什么内容或者已为最新

git branch -d&D 分支名称
删除&强制删除（如果还有未合并项）分支

git tag -a （版本名称）-m '版本信息'
查看版本记录（为当前分支添加版本号）
其作用为永久存储当前分支上的最新commit为一个不可修改的引用
标签的推送不能用push，需要git push origin <tagname>

#版本号也可以作为checkout的对象，进入分离头指针状态，修改将独立与任何分支，除非为此创建新分支，但这会使新分支移动，与标签不同。

git clone 远程仓库地址
将远程仓库（github）克隆到本地作为一个Git仓库

git push （远程仓库名，默认origin） （分支名，默认master）
上传当前master内容作为一个分支，后续需要pull request和merge

git pull （远程仓库名，默认origin） （分支名，默认master）
拉取远程仓库内容至本地作为一个独立未连接分支，需要merge
